Regular Expressions

Metacharacters - ^ $ . [ ] { } - ? * + ( ) | \

  • . - Any character - grep 'De..is' names.txt
  • ^ - Indicates regex should be at beginning of line - grep '^dkm' names.txt
  • $ - Indicates regex should be at end of line - grep 'dkm$' names.txt
  • [] - Match any single character w/in brackets - grep '[bg]zip' names.txt matches bzip* or gzip*. Metacharacters generally lose their special meaning w/in brackets, except:
  • [^bg] - When caret appears first within brackets, it negates the rest. - grep '[^bg]zip' names.txt matches .zip* except bzip* or gzip*. If the caret isn't first, it loses its special meaning and becomes part of the search string.
  • - - Used in brackets to indicate range - [A-z] or [A-Za-z0-9]. Put the dash first to take away its special meaning and make it part of the string.
  • ? - Match zero or one times.
  • * - Match zero or multiple times.
  • + - Match one or multiple times.
  • {n} - Match the element preceding brackets if it occurs exactly n times
  • {n,m} - Match the preceding element if it occurs at least n times, but no more than m times
  • {n,} - Match the preceding element if it occurs n or more times
  • {,m} - Match the preceding element if it occurs no more than m times
  • \ - Use to escape special meaning of characters
